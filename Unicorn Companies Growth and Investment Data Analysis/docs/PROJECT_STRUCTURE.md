## File Descriptions

### Data Files

- **Unicorn_Companies.csv**: Original dataset containing unicorn company information
- **Unicorn_Companies_cleaned.csv**: Cleaned and preprocessed dataset (generated by EDA scripts)

### Python Notebooks

1. **01_EDA_Unicorn_Companies.ipynb**: Exploratory Data Analysis
   - Data loading and inspection
   - Missing value analysis
   - Data cleaning and preprocessing
   - Basic visualizations

2. **02_Statistical_Analysis.ipynb**: Statistical Analysis
   - Descriptive statistics
   - Inferential statistics
   - Hypothesis testing
   - Correlation analysis

3. **03_Univariate_Bivariate_Multivariate_Analysis.ipynb**: Advanced Analysis
   - Univariate analysis
   - Bivariate analysis
   - Multivariate analysis
   - Comprehensive visualizations

4. **04_ML_Analysis.ipynb**: Machine Learning
   - Regression models (predict valuation)
   - Classification models (predict financial stage)
   - Model evaluation
   - Feature importance

### R Notebooks

1. **01_EDA_Unicorn_Companies.Rmd**: Exploratory Data Analysis (R version)
2. **02_Statistical_Analysis.Rmd**: Statistical Analysis (R version)
3. **03_Univariate_Bivariate_Multivariate_Analysis.Rmd**: Advanced Analysis (R version)
4. **04_ML_Analysis.Rmd**: Machine Learning (R version)

### Python Scripts

- **eda.py**: Standalone EDA script
- **univariate_bivariate_multivariate_analysis.py**: Standalone analysis script

### R Scripts

- **install_packages.R**: Script to install all required R packages

## Workflow

### Recommended Order of Execution

1. **EDA** (`01_EDA_Unicorn_Companies`)
   - Run first to clean and understand the data
   - Generates cleaned dataset

2. **Statistical Analysis** (`02_Statistical_Analysis`)
   - Requires cleaned dataset
   - Provides statistical insights

3. **Univariate/Bivariate/Multivariate Analysis** (`03_Univariate_Bivariate_Multivariate_Analysis`)
   - Requires cleaned dataset
   - Advanced relationship analysis

4. **Machine Learning** (`04_ML_Analysis`)
   - Requires cleaned dataset
   - Builds and evaluates ML models

## Output Files

### Models
- Saved in `results/models/` directory
- Can be loaded for predictions

### Plots
- Saved in `results/plots/` directory
- High-resolution PNG files

## Best Practices

1. **Always run EDA first** to generate cleaned dataset
2. **Use consistent random seeds** for reproducibility
3. **Save models and plots** for later reference
4. **Document findings** in notebooks
5. **Version control** code but not data (unless dataset is small)

## Notes

- The `results/` directory is gitignored by default (except .gitkeep files)
- Data files should not be committed if they are large
- Always respect dataset licensing terms
- Keep notebooks organized and well-documented
